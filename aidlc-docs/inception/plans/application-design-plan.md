### Q2 補足説明（違いの解説）

**A と B の違い**を具体的に説明します：

**選択肢A「ローカル検知 + Lambda 補助」**
```
PC上のPythonアプリ
  ├─ Transcribe からテキストを受信
  ├─ ローカルでキーワード検知（即時・低レイテンシ）← ここがポイント
  └─ Lambda にも送信（将来の高度な判断用）
```
- キーワード検知はPC内で完結 → **レイテンシ最小（<100ms）**
- Lambda はオプション的な役割（ハッカソンでは使わなくてもよい）
- ネットワーク断でも動作継続

**選択肢B「Lambda に全集約」**
```
PC上のPythonアプリ
  ├─ Transcribe からテキストを受信
  └─ Lambda に送信 → Lambda がキーワード検知 → PCに結果返却
```
- 全ての判断がクラウド側 → **往復レイテンシが発生（+200〜500ms）**
- ロジックの変更がLambdaだけで完結（コード管理が楽）
- ネットワーク断で動作停止

**ハッカソン向けの推奨**: レイテンシ要件（1秒以内）を考えると **A（ローカル検知）** が安全です。
ただし「AWSをフル活用している感」をデモで見せたい場合は B も選択肢です。
