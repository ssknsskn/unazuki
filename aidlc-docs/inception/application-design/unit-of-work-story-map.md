## ユニット別の機能カバレッジ

### Unit 1: PC Agent App
- FR-01（部分）: 音声キャプチャ → Transcribe → キーワード検知 → コマンド送信
- FR-02（部分）: 忖度ブーストキーワード検知・コマンド送信
- FR-03（部分）: 無発言タイマー管理・生存確認コマンド送信
- FR-04（部分）: 名前検知・コマンド送信
- FR-05（部分）: 相槌タイマー管理・相槌コマンド送信
- FR-06（部分）: ConfigManager による設定管理
- FR-08: PCアプリ全体のオーケストレーション
- FR-09（部分）: Config Web UI のスレッド起動・ホスト

### Unit 2: Device App
- FR-01（部分）: サーボモーターによるうなずき動作
- FR-02（部分）: 超高速うなずき・LED点滅演出
- FR-03（部分）: AWS Polly 音声再生（SORACOM Beam経由）
- FR-04（部分）: 超高速うなずき（名前検知時）
- FR-05（部分）: AWS Polly 音声再生（相槌ルーレット）
- FR-07: ハードウェア全制御（サーボ・LED・スピーカー）
- NFR-02（部分）: フォールバックモード（自律定期うなずき）

### Unit 3: Config Web UI
- FR-02（部分）: 忖度ブーストキーワードの管理UI
- FR-03（部分）: 無発言閾値の設定UI
- FR-04（部分）: 名前トリガーの管理UI
- FR-05（部分）: 相槌フレーズ・間隔の管理UI
- FR-06（部分）: 全設定項目のCRUD UI
- FR-09（部分）: ブラウザUI・REST API・テスト発火ボタン

## コンストラクションフェーズの実行順序

```
Unit 1: PC Agent App
  +-- 機能設計（FR-01, FR-02, FR-03, FR-04, FR-05, FR-08）
  +-- NFR要件（レイテンシ・安定性・Transcribeストリーミング）
  +-- NFR設計（ストリーミング処理パターン・再接続ロジック）
  +-- インフラ設計（AWS Transcribe設定・IAMロール・WebSocketポート）
  +-- コード生成（Unit 1 全モジュール）
  |
  v
Unit 3: Config Web UI
  +-- 機能設計（FR-06, FR-09）
  +-- NFR要件（即時反映・使いやすさ）
  +-- インフラ設計（FastAPIポート・静的ファイル配信）
  +-- コード生成（Unit 3 全モジュール）
  |
  v
Unit 2: Device App
  +-- 機能設計（FR-01, FR-02, FR-03, FR-04, FR-05, FR-07）
  +-- NFR要件（GPIO制御・SORACOM接続・フォールバック）
  +-- NFR設計（ハードウェア制御パターン・再接続ロジック）
  +-- インフラ設計（SORACOM Arc/Beam設定・GPIO ピン配置）
  +-- コード生成（Unit 2 全モジュール）
  |
  v
ビルド & テスト（全ユニット統合）
```
