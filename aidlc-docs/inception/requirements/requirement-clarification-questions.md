## 確認質問 1: 音声入力の実現方式

ご要望の「シンプル・即時性重視」を踏まえると、以下の2つが現実的な候補です。

**候補A: PCアプリ（常駐ソフト）でシステム音声をキャプチャ → AWS Transcribe にストリーミング**
- PCのシステム音声（会議ツールの出力音声）をリアルタイムでキャプチャ
- Python スクリプトが常駐し、AWS Transcribe Streaming API に直接送信
- Raspberry Pi はコマンドを受け取ってサーボを動かすだけ（軽量）
- ✅ 即時性が高い、ラズパイの負荷が低い
- ⚠️ PCにPythonスクリプトを常駐させる必要がある

**候補B: Raspberry Pi のマイクで直接集音 → AWS Transcribe にストリーミング**
- ラズパイにUSBマイクを接続し、会議音声を直接集音
- ラズパイ上のスクリプトが AWS Transcribe Streaming API に送信
- PCとラズパイが独立して動作（PCへのソフトインストール不要）
- ✅ セットアップがシンプル、PCに依存しない
- ⚠️ マイクの集音品質に依存、スピーカー音声の拾い方に工夫が必要

どちらが好みですか？

A) 候補A — PCアプリ常駐方式（即時性・精度重視）
B) 候補B — ラズパイマイク方式（PC依存なし・シンプル構成）
C) どちらでもよい（AIに任せる）
X) その他（[Answer]: タグの後に記述してください）

[Answer]: A
