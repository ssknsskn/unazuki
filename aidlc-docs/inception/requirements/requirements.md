# 要件定義書 — Unazu-kiro（全自動うなずきマシーン）

## インテント分析サマリー

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | ハッカソン向け「人をダメにするサービス」— ソフト＋ハードを組み合わせた全自動うなずきマシーン |
| **リクエストタイプ** | 新規プロジェクト（グリーンフィールド） |

## プロジェクト概要

**Unazu-kiro** は、オンライン会議中に「中身を聞かずに評価だけを得る」ための全自動うなずきデバイスです。クラウドAIが会議の音声をリアルタイム解析し、物理ロボット（Raspberry Pi + サーボモーター + お面）が「納得感のある頷き」を自動演出します。ユーモア溢れる「だめ人間」機能群を備え、ハッカソンのデモとして笑いと驚きを提供します。

---

## システム構成

```
[会議ツール（Zoom/Meet）]
        |
        | システム音声
        v
[PC常駐Pythonアプリ]
        |
        | AWS Transcribe Streaming API（リアルタイム文字起こし）
        v
[AWS Lambda / Bedrock（空気読みAI）]
        |
        | WebSocket / MQTT コマンド
        v
[Raspberry Pi（ハードウェアコントローラー）]
        |
        +---> [サーボモーター → お面が頷く]
        +---> [スピーカー → AWS Polly で発話]
        +---> [LED → 会議疲弊度インジケーター（将来拡張）]
```

## 機能要件

### FR-01: コアうなずき機能
- **説明**: 会議音声をリアルタイムで文字起こしし、発言が検知されたときにサーボモーターでお面を上下に動かす
- **入力**: AWS Transcribe からのトランスクリプトストリーム
- **出力**: Raspberry Pi へのサーボ制御コマンド
- **うなずきパターン**: ゆっくり（通常）、速め（重要発言）、超高速（忖度ブースト）

### FR-02: 忖度ブーストモード
- **説明**: 設定ファイルに登録されたキーワード（例：「社長」「重要」「予算」「決定」）を検知すると、デバイスがマッハで頷き始める
- **キーワード管理**: `keywords.yaml` または `keywords.json` 設定ファイルで自由に編集可能
- **動作**: キーワード検知 → 超高速うなずき（通常の3倍速）を3〜5秒間継続
- **視覚効果**: LEDが点滅して「忖度モード発動中」を演出

### FR-03: 生存確認フェイク
- **説明**: 一定時間（デフォルト60秒）自分の発言がない場合、AIが「あー、なるほど…」「確かに…」などの相槌をスピーカーから流し、寝落ちを隠蔽する
- **音声出力**: AWS Polly（TTS）で生成した音声をRaspberry Piのスピーカーから再生
- **発話タイミング**: 無発言60秒ごとにランダムな相槌フレーズを1つ再生
- **フレーズ例**: 「あー、なるほど…」「確かに…」「そうですね…」「うーん、なるほど」

### FR-04: 褒め言葉レーダー（自分の名前検知）
- **説明**: 設定ファイルに登録した自分の名前が呼ばれると、超高速でうなずく
- **設定**: `keywords.yaml` の `name_triggers` セクションで名前を登録
- **動作**: 名前検知 → 超高速うなずき（忖度ブーストと同様）を2秒間

### FR-05: ランダム相槌ルーレット
- **説明**: 一定間隔（デフォルト90秒）でランダムに「なるほど」「確かに」「おっしゃる通りです」などをスピーカーから発話
- **音声出力**: AWS Polly（TTS）
- **フレーズプール**: 設定ファイルで追加・編集可能
- **生存確認フェイク（FR-03）との違い**: FR-03は無発言トリガー、FR-05は時間間隔トリガー

### FR-06: キーワード設定管理
- **説明**: 忖度ブーストキーワード・名前トリガー・相槌フレーズを設定ファイルで管理
- **ファイル形式**: YAML（`config/keywords.yaml`）— バックエンドの永続化ストア
- **設定項目**:
  - `boost_keywords`: 忖度ブーストを発動するキーワードリスト
  - `name_triggers`: 褒め言葉レーダー用の名前リスト
  - `aizuchi_phrases`: ランダム相槌ルーレット用フレーズリスト
  - `silence_threshold_sec`: 生存確認フェイクの無発言閾値（秒）
  - `aizuchi_interval_sec`: ランダム相槌の間隔（秒）

### FR-09: ローカル設定Web UI
- **説明**: ローカルPC上でブラウザから設定を変更できる簡易Webダッシュボード
- **起動方法**: PCアプリ起動時に自動でローカルサーバーも起動（`http://localhost:8080`）
- **主な画面・機能**:
  - **キーワード管理**: 忖度ブーストキーワードの一覧表示・追加・削除
  - **名前トリガー管理**: 褒め言葉レーダー用の名前の追加・削除
  - **相槌フレーズ管理**: ランダム相槌ルーレット用フレーズの追加・削除・並び替え
  - **タイミング設定**: 無発言閾値・相槌間隔をスライダーまたは数値入力で調整
  - **デバイス接続状態**: Raspberry Pi との接続状態をリアルタイム表示（接続中 / 切断）
  - **テスト発火ボタン**: 各機能（うなずき・忖度ブースト・相槌）を手動でテスト実行
- **技術**: Python（FastAPI または Flask）+ シンプルなHTML/CSS/JS（フレームワーク不使用）
- **設定の保存**: 変更はリアルタイムで `config/keywords.yaml` に書き込み、動作中のアプリに即時反映
- **ハッカソン用途**: デモ直前にキーワードを素早く変更できること、審査員に見せて「こんな設定もできます」とアピールできること

### FR-07: ハードウェア制御
- **デバイス**: Raspberry Pi（メイン）
- **サーボモーター**: お面の首振り制御（GPIO経由、PWM制御）
- **スピーカー**: 3.5mmオーディオジャック or USB スピーカー
- **LED**: 忖度ブースト発動時の視覚演出（GPIO経由）
- **通信**: PCアプリからのコマンドをWebSocket または MQTT で受信

### FR-08: PCアプリ（常駐エージェント）
- **説明**: PCに常駐し、会議音声をキャプチャしてAWSに送信、AIの判断結果をRaspberry Piに転送する
- **音声キャプチャ**: システム音声（仮想オーディオデバイス経由）をリアルタイムキャプチャ
- **対応プラットフォーム**: macOS / Windows（ハッカソンではmacOS優先）
- **起動方法**: コマンドラインから `python main.py` で起動
- **対応会議ツール**: Zoom / Google Meet（プラットフォーム非依存、システム音声キャプチャのため）

## 非機能要件

### NFR-01: レイテンシ（即時性）
- 音声発言からうなずき動作開始まで **3秒以内**
- キーワード検知からブースト動作開始まで **1秒以内**

### NFR-02: 信頼性
- ハッカソンデモ中（最大30分）の連続動作を保証
- ネットワーク切断時はローカルフォールバック（定期うなずきモード）に自動切替

### NFR-03: セットアップ容易性
- `pip install -r requirements.txt` + `python main.py` で起動できること
- Raspberry Pi 側も `python device.py` 1コマンドで起動できること
- README に5分以内でセットアップできる手順を記載

### NFR-04: コスト
- AWSの無料枠・低コスト範囲内で動作（ハッカソン期間中の費用が数百円以内）
- Amazon Transcribe Streaming: 従量課金（デモ用途では問題なし）
- AWS Polly: 無料枠内（月500万文字まで無料）

### NFR-05: ユーモア・デモ映え
- デバイスの動作が「笑える」「驚ける」ものであること
- お面の首振りが視覚的にわかりやすいこと（振り幅・速度の調整可能）
- ハッカソン審査員が「これは面白い」と感じるユーモアを持つこと

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| **音声認識** | AWS Transcribe Streaming API |
| **AI判断** | AWS Lambda（キーワード検知・ロジック） + Amazon Bedrock（オプション：高度な空気読み） |
| **TTS（音声合成）** | AWS Polly |
| **PCアプリ** | Python 3.11+（sounddevice / pyaudio + boto3） |
| **ハードウェア** | Raspberry Pi 4 / 5（推奨） |
| **サーボ制御** | RPi.GPIO / pigpio（PWM） |
| **PC↔デバイス通信** | WebSocket（`websockets` ライブラリ）または MQTT（`paho-mqtt`） |
| **設定管理** | YAML（`PyYAML`） |
| **設定Web UI** | FastAPI + HTML/CSS/JS（フレームワーク不使用） |
| **会議ツール** | Zoom / Google Meet（システム音声キャプチャ、API不使用） |
